media control software packages

You can send signals from boogiepants to any software that receives OSC signals. Theoretically you could control a robot arm that way, or target a bomb, or whatever. But boogiepants is designed for performance with sound.

There are a number of software packages you can use with boogiepants:

  • Pure Data is nice. Both powerful and free. We package a PD patch with boogiepants, which illustrates many useful techniques.
  • Max/Msp
  • supercollider
  • and many others.

running the other software elsewhere on the network

You can run boogiepants on a separate machine from the one controlling the media:

  1. make sure the network is set up and connected;
  2. determine the IP address of the machine the media control software is running on. The way to do this will depend on the operating system the other machine is running:
    • If the remote machine is a Mac, go into System Preferences, click on Network, click on the Advanced button, and choose the 'TCP/IP' tab from the top of the window. You probably want to use the IpV4 address. Or for the non-command-line-phobic, enter ifconfig | grep inet in Terminal;
    • If the remote machine is a Windows Vista machine, go into Control Panel, click on Network and Sharing Center. Click on View Status, and then the Details button. You probably want to use the IPv4 address. Or for the non-command-line-phobic, use the ipconfig command;
    • If the remote machine is running Linux, use the ifconfig command;
  3. Determine the port it is listening for OSC signals on. That will depend on the software package you are using;
  4. on the boogiepants control panel, under 'OSC endpoint,' enter the IP you found;
  5. on the boogiepants control panel, under 'OSC port,' enter the port number you found;
  6. hit enter.

If the network is operational, boogiepants should send signals across to the other machine. If you do it over wi-fi, it may be erratic.
